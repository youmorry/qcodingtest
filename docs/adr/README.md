# アーキテクチャ決定記録（ADR）

技術的な決定事項を1行ずつ記録する。新しい決定は末尾に追記する。

| # | 日付 | 決定 | 理由 |
| --- | --- | --- | --- |
| 1 | 2026-06-27 | 文字列カラムは `varchar(n)` ではなく `text` を採用し、文字数などの入力制約はドメイン層で検証する | PostgreSQL では両者に性能差がなく、`varchar(255)` 等の長さは根拠の薄い慣習値。入力制約はドメイン層で検証する方針（README 2.4）と一貫させる |
| 2 | 2026-06-27 | 中間テーブルは `author_books`、複合主キーを `(author_id, book_id)` とする | 唯一のリレーション参照系 `GET /api/authors/{authorId}/books`（著者→書籍）が主キーインデックスだけで完結する |
| 3 | 2026-06-27 | 中間テーブルに書籍→著者方向（`book_id`）の補助インデックスは作らない | 存在チェックは各テーブルのPK、重複防止は複合PKで足り、`book_id` を使うのは低頻度の書き込みのみ。参照系の要件もなく過剰実装となるため |
